    hexo+github搭建个人博客

    发布时间 : 2024-05-09 16:39
    阅读 :

    Hexo：一个简单的开源博客框架，支持Markdown语言
    GitHub Pages：由GitHub提供的静态站点托管服务。通过托管在Github仓库的博客代码发布静态网站页面。

    准备工作

    注意

    • 该流程基于win10操作系统

    安装Git

    前往Git官网下载并安装
    因为需要使用Git Bash，所以下载setup版本

    • 安装位置可自定义
    • 组件默认即可
    • 设置环境变量的选项里，选择第一个 Use Git from Git Bash only

    安装Node.js

    前往Node.js中文网站下载并安装
    安装完成后打开集成终端（Git Bash 或者 cmd）
    依次输入

    node -v
    npm -v
    git -version

    出现版本号即为安装成功
    强烈提醒
    配置cnpm，或者配置淘宝镜像，不然下载依赖的速度将会非常慢

    • 配置cnpm

      npm install -g cnpm –registry=https://registry.npmmirror.com

    • 直接修改npm

    临时使用

    npm –registry https://registry.npmmirror.com

    永久使用

    npm config set registry https://registry.npmmirror.com

    恢复原网址

    npm config set registry https://registry.npmjs.org

    安装hexo

    确认nodejs安装成功后

    npm install hexo-cli -g

    耐心等待安装完成

    初始化博客

    新建博客的本地文件夹，在文件夹内右键，启用git bash，并输入

    hexo init #初始化

    也可以在父文件夹启用git bash，直接新建博客文件夹

    hexo init <文件夹名称>

    初始化完成后安装组件

    npm install #安装组件

    组件安装完成后就可以在本地部署和运行博客了

    hexo g #生成页面
    hexo s #本地预览

    连接GitHub

    注册Github账户，注册邮箱

    system级的设置会保存在Git的安装目录etc\gitconfig中
    global级的设置会保存在C盘的用户文件夹下的gitconfig中
    ssh秘钥会保存在c盘的用户文件夹下的gitconfig中

    注意不要把以上文件夹放入任何项目文件中，避免因为上传而泄露密码！！！！

    启动Git Bash，设置GitHub的用户名和邮箱

    git config –global user.name “GitHub用户名”
    git config –global user.email “GitHub邮箱”

    两种保存密码的方式

    1. 直接利用git保存密码 不建议

    如果在安装Git时安装了GitGUI，会自动在system级别中设置credential.helper为manager，会默认使用该方式

    • –system 为系统级设置，该计算机系统下所有用户有效不建议使用
    • –global 为用户级设置，该用户下所有项目有效
    • 在项目仓库文件中直接 git config，为项目级，仅在当前项目有效

    设置git记录账户密码

    git config –global credential.helper manager

    设置git取消（删除）记录的密码

    git config –global –unset credential.helper

    查询凭证的存储模式

    git config –global credential.helper
    或
    git config –global –list
    或
    git config –global -l

    通过修改配置文件设置记录

    git config –file=~/.git_credentails –global credential.helper “manager”
    或
    git config –global user.name “manager”

    临时记住默认15分钟

    git config –global credential.helper cache

    记住1小时

    git config credential.helper ‘cache –timeout=3600’

    凭证储存模式

    • cache 临时保存凭证
    • store 永久性明码保存在本地磁盘中
    • manager 利用windows的凭证保存功能，加密存放在磁盘中，cmd命令行和控制面板都可删除
    • osxkeychain 适用于Mac用户，与manager同理

    2. SSH秘钥
    查询用户主目录下是否有.ssh
    有的话直接跳过这一步
    在git bash中输入

    ssh-keygen -t rsa -C “GitHub邮箱”

    打开.ssh文件夹，打开公钥id_rsa.pub并复制里面的内容
    登录GitHub，进入Setting页面，找到SSH and GPG keys页面，点击New SSH key，Title自定义一个，黏贴公钥内容，点击Add SSH key

    在git bash中输入

    ssh -T git@github.com

    一路确认到最后出现成功连接的提示。

    创建Github Pages仓库

    1. 在GitHub主页点开New repository
    2. Repository name 输入 用户名.github.io 必须用用户名
    3. 勾选 “Initialize this respository with a README”
    4. Description选填
    5. 点击Create repository

    博客地址为： https://用户名.github.io

    部署博客

    在博客文件夹的git终端输入

    hexo d

    常用hexo命令

    hexo new “name” # 新建文章
    hexo new page “name” # 新建页面
    hexo g # 生成页面
    hexo d # 部署
    hexo g -d # 生成页面并部署
    hexo s # 本地预览
    hexo clean # 清除缓存和已生成的静态文件
    hexo help # 帮助
